## 📌 Project Overview

This project performs **sales and dealer order analysis using SQL**.
A relational database was created to store information about **dealers, products, and orders**, and SQL queries were used to extract meaningful insights from the data.

The project demonstrates **database design, data insertion, joins, aggregation functions, and analytical queries**.

## 🗂 Database Structure

The project contains **three tables**:

| Table Name       | Description                              |
| ---------------- | ---------------------------------------- |
| dealers_customer | Dealer information and shop details      |
| product          | Product details including price and type |
| orders           | Orders placed by dealers                 |

### Relationship

Dealers place orders for products.

```
Dealers → Orders → Products
```

---

## 📊 SQL Analysis Performed

The following analysis queries were executed:

* View product names and prices
* Calculate **total revenue**
* Find **average product price**
* Identify **maximum and minimum order values**
* Find **cities with multiple dealers**
* Calculate **revenue by product type**
* Identify **most expensive products**
* Detect **high value orders**
* Join multiple tables to analyze order details

---

## 📈 Key Insights

From the SQL analysis:

* **Cup-type products generated the highest revenue**
* Some orders exceed **₹1000**, indicating bulk purchases
* Cities like **Chennai and Pune have higher dealer concentration**
* **Blueberry** is the most expensive product in the box category
